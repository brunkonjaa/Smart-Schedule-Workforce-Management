# Report Diagrams

These six diagrams give a visual version of the final system. The PNG files are ready for the report and the matching SVG files are the editable source generated by `scripts/generate_report_diagrams.py`.

1. `01_use_case` - final manager, staff and shared use cases with frozen exclusions
2. `02_complete_erd` - the live PostgreSQL tables through migration `027`, including the session and migration tables
3. `03_system_architecture` - browser/PWA, Render Node process, Express/WebSocket layers, Neon and Brevo
4. `04_assignment_conflict_sequence` - validation, serializable checks, hard limits, warning and audit response
5. `05_shift_swap_state` - live request/accept/manager decision path and database-only statuses
6. `06_nodychat_unread_flow` - session, participant, message, read state and broadcast checks

`availability_entries` is not drawn as a live table because migration `014_remove_weekly_availability.sql` removes it. The diagrams also avoid showing a separate recommendation engine, payroll service, notification service or message broker because those do not exist in this repo.
